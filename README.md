# CLI Agent с веб‑интерфейсом

Автономный помощник на Rust, сочетающий командную строку (CLI) и веб‑интерфейс. Агент имеет доступ к локальной системе (выполнение команд) и к сети (веб‑поиск через DuckDuckGo и GitHub). Взаимодействие возможно как через терминал, так и через современный веб‑чат, работающий на локальном сервере.

## Возможности

- **Выполнение локальных команд**  
  Вывод в формате `[RUN: команда]`. Агент выполнит команду и вернет результат.
- **Веб‑поиск**  
  Запрос в формате `[WEB_SEARCH: запрос]`. Сначала проверяется публичный репозиторий GitHub (если в запросе есть `github.com/`), затем используется DuckDuckGo Lite.
- **Потоковый вывод**  
  Ответы большой языковой модели транслируются в реальном времени как в терминал, так и в веб‑интерфейс через WebSocket.
- **Веб‑интерфейс**  
  Современный одностраничный чат на Tailwind CSS и Lucide иконках, доступный по адресу `http://127.0.0.1:8080`.
- **Гибкая настройка**  
  Легко менять модель, токен API, адрес Ollama и другие параметры через константы в `src/main.rs` или переменные окружения.
- **Корректное завершение работы**  
  При вводе команды `exit` в CLI веб‑сервер получает сигнал shutdown, отправляет прощальное сообщение и корректно завершает работу.
- **Автономное исследование перед рекомендациями**  
  При запросе советов/рекомендаций агент сначала собирает базовый контекст проекта (список файлов, `Cargo.toml`, структуру исходников, README) и только потом формирует ответ.

## Требования

- Rust stable (тестировалось на 1.70+)
- Доступ к API Ollama (например, https://ollama.com) с действующим токеном
- Подключение к интернету (для веб‑поиска)
- Операционная система: Linux/macOS (Windows поддерживается частично через `cmd.exe`)

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/koltchenkoelena-boop/cli-agent.git
   cd cli-agent
   ```

2. Убедитесь, что у вас установлен Rust и Cargo:
   ```bash
   rustc --version
   cargo --version
   ```

3. **Настройте токен API Ollama через переменную окружения** (рекомендуемый способ):
   ```bash
   export OLLAMA_API_TOKEN=ваш_токен_здесь
   ```
   Добавьте эту строку в ваш `~/.bashrc`, `~/.zshrc` или запустите её перед запуском программы.

   Альтернативно, если вы предпочитаете хардкод токена в коде, вы можете отредактировать `src/main.rs` и изменить инициализацию `AUTH_TOKEN` на строковый литерал. Однако использование переменной окружения более безопасно и гибко.

4. Соберите проект:
   ```bash
   cargo build --release
   ```
   Бинарный файл появится в `target/release/cli_agent`.

## Запуск

```bash
OLLAMA_API_TOKEN=ваш_токен_здесь ./target/release/cli_agent
```

Вы увидите что-то вроде:

```
CLI Agent (Web + CLI) — Web UI на http://127.0.0.1:8080
Введите 'exit' для выхода
```

Откройте браузер и перейдите на http://127.0.0.1:8080 — там будет доступен чат‑интерфейс.

### Использование в CLI

- Введите запрос и нажмите Enter.
- Чтобы выполнить локальную команду, начните строку с `[RUN: ` и завершите её `]`.
  Пример: `[RUN: ls -la]`
- Чтобы запустить веб‑поиск, используйте `[WEB_SEARCH: ваш запрос]`.
  Пример: `[WEB_SEARCH: последние новости о Rust 2024]`
- Для завершения работы введите `exit`. Агент корректно остановит веб‑сервер и выйдет.

### Использование через веб‑интерфейс

- Введите сообщение в поле ввода и нажмите Enter (или нажмите кнопку отправки).
- Shift+Enter — новая строка без отправки.
- История навигации: клавиши ↑ и ↓.
- Индикатор «печати» показывает, когда агент генерирует ответ.
- Системные сообщения (например, о подключении/отключении веб‑сокета, выполнении команд) отображаются в чате.

## Конфигурация

В `src/main.rs` находятся следующие константы, которые можно изменить под свои нужды:

| Константа            | Описание                                                |
|----------------------|---------------------------------------------------------|
| `API_URL`            | Endpoint API чата Ollama (по умолчанию `https://ollama.com/api/chat`) |
| `AUTH_TOKEN`         | **Теперь читается из переменной окружения `OLLAMA_API_TOKEN`** (см. выше) |
| `MODEL`              | Имя используемой модели (по умолчанию `nemotron-3-super:cloud`) |
| `DDG_LITE_URL`       | URL DuckDuckGo Lite для веб‑поиска                     |
| `DDG_USER_AGENT`     | User‑Agent, используемый при запросах к DDG и GitHub    |
| `MAX_SEARCH_CHARS`   | Максимальная длина текста результата поиска (символов) |
| `TEMPERATURE`        | Температура генерации (0.0 – детерминировано, 1.0 – креативно) |
| `SYSTEM_PROMPT`      | Системный промпт, задающий поведение агента (включает правила форматирования и автономного исследования) |

## Как это работает

1. При запуске создаётся `tokio`‑runtime.
2. Инициализируется слой трассировки (`tracing`) для логирования в stderr.
3. Создаётся HTTP‑клиент с заголовком авторизации для обращения к Ollama (значение берётся из переменной окружения `OLLAMA_API_TOKEN`).
4. Запускается фоновый веб‑сервер (Axum) на `0.0.0.0:8080`, обслуживающий:
   - WebSocket эндпоинт `/ws` для обмена событиями (`FrontendEvent` и `ClientCommand`);
   - Статические файлы из директории `static/` (главная страница `index.html`).
5. В отдельном блокирующем задаче (`tokio::task::spawn_blocking`) запускается `rustyline` для чтения строк из терминала.
6. Основной цикл использует `tokio::select!` для ожидания трёх источников событий:
   - ввод из командной строки (`cli_rx`);
   - команды из веб‑интерфейса через `mpsc` (`cmd_rx`);
   - сигнал прерывания `Ctrl+C`.
7. При получении запроса (как из CLI, так и из WS) вызывается `process_task`, которое:
   - Добавляет сообщение пользователя в историю диалога.
   - В цикле обращается к модели Ollama через потоковый запрос.
   - Во время стрима выводит части ответа в терминал и буферизирует их для отправки в веб‑интерфейс (по достижению порога или знаку препинания).
   - Если в ответе встречается шаблон `[RUN: ...]` или `[WEB_SEARCH: ...]`, агент выполняет соответствующее действие, отправляет промежуточные события (`ToolExecuting`, `ToolResult`) и продолжает диалог с учётом полученного вывода.
   - Когда модель завершает генерацию без специальных тегов, цикл прерывается и управление возвращается основному циклу.
8. При получении команды `exit` из CLI агент отправляет сигнал остановки веб‑серверу, ждёт короткой задержки и завершает работу.
